Cognitive Walkthrough Protocol for Mobile Itinerary Builder Evaluation

Goal

Evaluate how easily a first-time or infrequent user can create and manage a travel itinerary using the mobile application.

Step 1: Entry into the App

Task: Open the app and locate the trip planner

Evaluator Questions:

  • Will the user know that building an itinerary is possible here?
  • Is it obvious where to start (e.g., "Plan a Trip," "Itinerary," "Trips")?
  • Does the label/icon for the itinerary builder match the user’s mental model?
  • Would a new user think to tap here?

Step 2: Starting a New Itinerary

Task: Begin creating a new itinerary

Evaluator Questions:

  • Is it clear how to create a new trip (button, CTA, FAB)?
  • Does the system provide enough context about what an itinerary or a tip planner is?
  • Will the user recognize what information they need to provide first (e.g., destination, dates)?
  • Does the flow reduce hesitation for someone just exploring (not yet committed)?

Step 3: Entering Destination and Dates

Task: Add the trip destination and travel dates.

Evaluator Questions:

  • Is it obvious that destination and dates are required starting points?
  • Will the user know what format is expected for dates?
  • Does the interface handle flexible dates or "undecided" inputs gracefully?
  • Is there feedback that the system has understood their entry?

Step 4: Adding Activities, Lodging, or Events

Task: Search for and add items (attractions, restaurants, tours, events, lodging).

Evaluator Questions:

  • Is it clear how to search for and add activities?
  • Will the user understand the difference between viewing details vs. adding to itinerary?
  • Is feedback provided after adding an item (e.g., confirmation, visual change)?
  • Will the user know if they can add custom notes/activities not in the database?
  • Are there smart defaults for inputs?

Step 5: Organizing the Itinerary

Task: Arrange activities by day and time.

Evaluator Questions:

  • Is it clear how to reorder or move activities between days?
  • Will users notice drag-and-drop, or is there another clear interaction pattern?
  • Is there visibility into time conflicts (overlapping events, travel time)?
  • Does the system guide users toward a realistic schedule without overwhelming them?

Step 6: Editing and Customizing

Task: Modify an existing item (e.g., change time, remove an activity, add a note).

Evaluator Questions:

  • Is the edit option discoverable from the itinerary view?
  • Will the user understand what can be edited (time, notes, duration)?
  • Is there a clear way to remove or undo actions without fear of losing progress?
  • Do customization options align with how travelers think about planning (flexibility vs. structure)?

Step 7: Saving and Exiting

Task: Save the itinerary and exit the builder.

Evaluator Questions:

  • Will the user know when/if their itinerary is automatically saved?
  • Is there a clear confirmation that their work won’t be lost?
  • Is the difference between "save," "share," and "publish" clear?
  • Can they easily return to this itinerary later?

Step 8: Sharing or Collaborating

Task: Share the itinerary with travel companions or access it across devices.

Evaluator Questions:

  • Is the sharing/collaboration option visible and understandable?
  • Will the user know the difference between view-only vs. edit permissions?
  • Does the app clarify how companions will receive or access the itinerary?
  • Is it easy to undo or manage permissions?

Step 9: Returning to an Existing Itinerary

Task: Locate and re-open a previously saved itinerary.

Evaluator Questions:

  • Will the user know where to find their saved itineraries?
  • Is it easy to distinguish between multiple trips?
  • Will they feel confident that nothing has been lost or misplaced?
  • Are itineraries synced and accessible across platforms (if applicable)?

Wrap-Up

After walking through all steps, evaluators should summarize:

  • Where the user might struggle with discoverability, feedback, or mental model mismatches.
  • Whether the flow supports both exploratory planners (early browsing) and structured planners (detailed scheduling).
  • What features are hidden or missing?
  • What features seem distracting or not needed?
  • Would our specific audience find this tool usable and valuable?
