Power Grid Event Investigation

Summary

Collaborating with field technicians, grid operators, and data scientists, I developed investigation tools for monitoring the electrical grid, city utitlites, and security cameras. These tools include a customizable dashboard, AI-powered metrics, and contextual data visualizations.

Background

Power grids are complex systems with thousands of components that require constant monitoring. Grid operators need to efficiently manage these systems by focusing on anomalies and potential issues rather than monitoring normal operations. An approach known as "management by exception."

Research and Discovery

I gathered insights through a combination of competitive analysis and interviews with field technicians and grid operators.

  • Current tools force them to be reactive rather than proactive
  • Lack of context makes it difficult to prioritize issues
  • Sending crews to fix equipment is expensive

Content Inventory

  • Lack of a strategic content structure led to a fragmented user experience as information accumulated over time
  • Deeply nested tabs obscured critical information, hindering efficient user navigation and task completion
  • Redundant content further complicated the interface, leading to user confusion and decreased productivity

Structure Evolution

Presented distinct architectural options with clear trade-offs, incorporating stakeholder input (incl. the CEO), to foster collaborative discussion and guide the organization toward an evidence-based decision.

Data Science Collaboration

  • Defined key metrics for different types of grid events
  • Identified machine learning abilites for predictive notifications
  • Discoved opportunites for automated anomaly detection

Challenge

Original requirements too large project scope, poorly defined priorities, and an unrealistic timeline.

Specific features requested by only one customer, raising concerns about their broader impact and feasibility.

Visualize the Challenge

Mapped each requirement/feature from the list directly onto the relevant steps within the core user journey.

Used the visual flow+requirements map as a tangible tool to lead a structured discussion on feature prioritization based on user impact at each stage of the journey.

Strategic Questioning

  • Feature (positive/negative) on the wider user base
  • Effect on the project timeline
  • Importance of customer

The Solution

Intelligent Customizable Dashboard

  • Drag-and-drop prioritized arrangement of metrics
  • Aggregated data showing grid-wide health
  • AI-powered metrics replacing manual thresholds

Patterns for Accessiblity

Enhanced Event Investigation

Open
  • Ability to explore high-level categories
  • Filters designed to isolate specific events and reveal detailed insights
  • Contextal metrics (current, temperature) telling a more complete story

Waveform Analysis

  • Enabled operators to drill down into granular details with interactive visualizations, clarifying complex grid events
  • Overlaying waveforms made it easier to spot trends and detect anomalies
  • Offered data export capabilities to support in-depth analysis and integration with other tools

Outcome

Enhanced Decision Support

Aggregated sensor data validated issues and improved notification accuracy, reducing noise by 37%

Improved Visualization

Trend and pattern visualizations allowed operators to identify potential issues nearly twice as fast

Proactive Issue Management

Faster response times and earlier insights enabled proactive maintenance

Cost-Effective Maintenance

The improvements fostered confident decisions, lowering maintenance expenses and increasing equipment lifespan
